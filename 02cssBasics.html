Aufbau eines Stylesheets

Grundsätzlich sollte ein Stylesheet so aufgebaut sein, daß die Spezifitätsregeln ungefähr die sequentielle Abarbeitung abbilden. D.h. Selektoren mit niedriger Spezifität kommen an den Anfang des Stylesheets und solche mit höchster Spezifität werden am Ende notiert.

Allgemeine Regeln und vererbbare Proerties werden zu Beginn des Stylesheets vereinbart, so daß sie teils an späterer Stelle durch spezifischere Regeln überschrieben werden können.

Block- und Inlineelemente

Jedes Element ist per default eine rechteckige Box mit folgenden Bestandteilen:

  • content (width und height)
  • padding (Innenabstand)
  • border (Rahmen)
  • margin (Außenabstand)

Blockelemente nehmen per default 100% des Elternelementes ein.

Inline-Elemente nehmen nur den tatsächlich benötigten Raum ein und lassen andere Elemente neben sich zu. Sie können keine width und keine vertikalen margin erhalten.

Ein Inline-Block-Element kann das property width bekommen und vertikale margin und es kann andere Elemente neben sich erlauben.

Das box-sizing Property

Das box-sizing ist die Grundlage für die Größenberechnung von Blockelementen. Es stehen u.a. die folgenden zwei Werte zur Verfügung:

  • border-box
  • content-box

Größenberechnung

border-box
tatsächliche Breite = width + 2 * margin

content-box
tatsächliche Breite = width + 2 * margin + 2 * border + 2 * padding

Das box-sizing sollte für alle Elemente gleich sein und wird deshalb im Universal Selektor notiert. So kann es später auch für einzelne Elemente überschrieben werden.
